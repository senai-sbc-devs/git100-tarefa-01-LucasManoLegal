Fluxo detrabalho do Git e GitHub

Este é um site simples com o objetivo de explicar o fluxo de trabalho com o Git e GitHub.

Clone

Definição:

Clonar um repositório Git significa criar uma cópia exata do repositório remoto em sua máquina local.

Como Usar:

O comando básico para clonar um repositório é git clone URL_do_repositório. Isso cria uma cópia local do repositório em seu computador, permitindo que você trabalhe nele.

Branch

Definição:

Um branch é uma ramificação separada no histórico do repositório, permitindo que você trabalhe em novas funcionalidades ou correções de bugs sem interferir no código principal.

Como Usar:

Para criar um novo branch, você usa o comando git branch nome_do_branch. Para mudar para um branch existente, use git checkout nome_do_branch ou, a partir do Git 2.23, git switch nome_do_branch.

Commit

Definição:

Um commit é um ponto no histórico do repositório que armazena as alterações feitas nos arquivos. Cada commit tem uma mensagem associada que descreve as mudanças realizadas.

Como Usar:

Após realizar alterações nos arquivos, você usa git add nome_do_arquivo para preparar as mudanças e git commit -m "Mensagem do commit" para confirmar essas alterações.

Merge

Definição:

Merging é o processo de combinar as alterações de diferentes branches. Isso é geralmente feito para incorporar o trabalho de um branch de recurso de volta ao branch principal.

Como Usar:

Após concluir o trabalho em um branch, você pode mesclá-lo de volta ao branch principal usando git merge nome_do_branch.

PullRequest

Definição:

MEm sistemas como o GitHub, GitLab ou Bitbucket, um pull request (também conhecido como merge request) é uma solicitação para mesclar as alterações de um branch para outro. Isso é comumente usado em colaboração, permitindo que outros revisem e comentem sobre as mudanças antes que sejam mescladas.

Como Usar:

Após concluir as alterações em um branch, você cria um pull request na plataforma de hospedagem do seu repositório. Outros membros da equipe podem revisar as mudanças antes de serem mescladas no branch principal.
